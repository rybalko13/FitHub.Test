## О продукте

**FitHub** — веб-приложение, единый хаб для данных о здоровье и фитнесе.
Подключаешь свои устройства и приложения (Strava, FatSecret, Apple Health и др.),
и видишь всё на одном экране: тренировки, питание, сон, восстановление.
Плюс ИИ-рекомендации на основе связки этих данных.

**Версия:** 1.0 — MVP
**Статус:** в разработке
**Платформа:** веб (первый этап), мобильное приложение — потом

### Ключевая боль, которую решаем

Пользователь держит 3–5 приложений (часы, дневник питания, трекер сна), но они
не «разговаривают» друг с другом. Никто не даёт единого вывода вида
«ты в дефиците 700 ккал — это хорошо или плохо для твоих целей?».

### Главное отличие от конкурентов

Garmin Connect, Apple Health, Google Fit — закрытые экосистемы, работают только
со своими устройствами. **FitHub работает с любыми устройствами одновременно.**

## Целевая аудитория

### MVP — B2C

Любитель фитнеса 20–40 лет. Тренируется регулярно, ведёт дневник питания,
носит умные часы. Цель: понимать своё тело и прогресс в одном месте.

### После MVP — B2B

Персональные тренеры. Хотят видеть прогресс нескольких клиентов в одном месте.
Монетизация через тариф Coach. **В MVP не включается** — сначала рабочий продукт
для обычных пользователей, потом B2B.

## Функционал MVP

1. **Главный дашборд** — всё на одном экране: калории (потреблено vs сожжено),
   тренировки, сон, восстановление, макросы, прогресс к цели.
2. **ИИ-рекомендации** — на основе реальных данных пользователя
   («ты недоедаешь в дни тренировок», «восстановление низкое 3 дня — снизь нагрузку»).
3. **Еженедельный отчёт** — автоматически на email: сводка + топ-3 инсайта + рекомендации.
4. **Ручной дневник тренировок** — для тех, кто не синхронизирует часы.

**Что НЕ входит в MVP:**
- B2B-функционал для тренеров (тариф Coach)
- Мобильное приложение
- Расширенный ИИ с прогнозами и персональными планами
- Интеграции с MyFitnessPal, Yazio, Cronometer, Whoop

## Интеграции — приоритеты

### В MVP

| Сервис | Тип данных | Приоритет |
|--------|------------|-----------|
| Strava | Тренировки, кардио, GPS | 1 |
| FatSecret | Питание, калории, макросы | 1 |
| Apple Health | Активность, сон, ЧСС | 2 |
| Google Fit | Активность, шаги, тренировки | 2 |
| Fitbit | Сон, ЧСС, шаги, восстановление | 3 |

### Отдельный случай — Garmin

У Garmin нет открытого API для сторонних разработчиков. В MVP реализуем
**ручной импорт**: пользователь раз в неделю загружает CSV/FIT-файл из
Garmin Connect. Автопарсинг через браузер — опциональная идея,
но она может нарушить TOS Garmin, поэтому по умолчанию не делаем.

### После MVP

MyFitnessPal, Yazio, Cronometer, Whoop — по мере роста базы.

## Алгоритм работы системы

1. Пользователь подключает аккаунты через OAuth.
2. Система раз в день (ночью) забирает данные через API каждого сервиса.
3. Данные нормализуются — приводятся к единому формату в базе.
4. ИИ анализирует связку данных и генерирует инсайты.
5. Пользователь видит дашборд с актуальными данными.
6. Раз в неделю — автоматический отчёт на email.

## Бизнес-модель

| Тариф | Цена | Что включено |
|-------|------|--------------|
| Free | 0 ₽ | 2 интеграции, базовый дашборд, данные за 14 дней |
| Pro | ~500 ₽/мес | Все интеграции, ИИ-рекомендации, еженедельный отчёт на email, полная история |
| Coach | ~1500 ₽/мес | Всё из Pro + до 10 клиентов, тренерский дашборд (после MVP) |

### Механики конвертации Free → Pro

- **Обрезка истории** — Free видит только 14 дней. Метрики и прогресс есть, но история обрезается — человек сам чувствует нехватку.
- **Пример отчёта** — Free-пользователь видит демо-версию еженедельного отчёта (заблюренную или с фиктивными данными) + кнопку «получить свой».
- **Один реальный отчёт** — через 2–3 недели после регистрации Free-пользователь автоматически получает один настоящий ИИ-отчёт по своим данным. Письмо заканчивается: «Хочешь так каждую неделю? Перейди на Pro». Это *taste of value* — человек видит реальную пользу, а не рекламу.

## Как работать с Владом — правила (только для FitHub)

> Эти правила действуют **только** когда Claude работает с проектом FitHub.
> В других проектах Влада свои настройки — сюда не подглядывать.


### Язык и стиль

- Всегда по-русски.
- Обращаться по имени — **Влад**.
- Тон — товарищеский, без официоза.
- Влад — новичок в программировании. Объяснять простыми словами, сначала идея,
  потом код. Никаких технических терминов без пояснения.
- Если Влад спрашивает «почему» — объяснять глубже.

### Нон-конформизм (важно!)

- Спорить, если Влад неправ. Не соглашаться просто так.
- Предлагать альтернативы.
- Не быть yes-man. Если идея плохая — сказать прямо.
- Если решение неоптимально — указать на это и обосновать.

### Подтверждения действий

- Перед каждым значимым действием (создание файла, удаление, изменение структуры)
  спрашивать подтверждение.
- Формат: **«Я собираюсь сделать X. Причина: Y. Продолжаем?»**
- Не выполнять несколько шагов подряд без одобрения каждого.

### Код

- Перед написанием — объяснить что и почему именно так.
- **Никаких лишних зависимостей.** Прежде чем добавить библиотеку — проверить,
  можно ли обойтись встроенными средствами.
- **Никаких дублирующих функций.** Если похожая логика уже есть — переиспользовать.
- Код простой и читаемый, не умный ради умного.

### Обязательный чеклист после любого кода

После написания кода показывать Владу этот чеклист с отметками:

- [ ] Нет дублирующих функций или логики
- [ ] Нет лишних импортов и зависимостей
- [ ] Каждая функция делает одно дело
- [ ] Код делает именно то, что просил Влад — не больше и не меньше
- [ ] Нет хардкода там, где должны быть переменные

### Структура ответа

1. Что собираюсь делать.
2. Код или действие.
3. Результат чеклиста + вопрос **«Всё верно, продолжаем?»**
